# Contributing to B3YOND Co.

This repository is currently maintained by the founder.

## Documentation Standards

- Keep writing clear and direct.
- Record major decisions with dates.
- Do not delete old decision history. Add new entries instead.
- Use Markdown for all living documentation.
- Keep sensitive information out of the repository.

## Commit Style

Use clear commit messages:

```text
docs: update founder notebook
docs: add invention log entry
brand: update color notes
legal: add trademark research note
roadmap: revise milestone plan
```

## Branches

- `main` should remain stable.
- Use feature branches for major changes.

Example:

```text
docs/founder-letter-update
brand/logo-directions
legal/trademark-research
```
